Who it's for

Account managers, sales reps, and non-technical managers who deal with clients, proposals, and relationship history. If you've ever spent an hour digging through emails before a client call, this workshop is for you. No coding experience required.

What you'll build

account — a CLI that ingests client emails, call notes, and meeting transcripts, maintains a per-account SQLite ledger, generates structured briefs before every client call, and drafts tailored proposals from RFPs.

Format options

Combined: 4 hours (2 modules + break)
Standalone: Module 1 or Module 2 as a 2-hour session

What to bring

Claude Pro · Python 3.11+ · a laptop that runs terminal commands

› customer-intel  ·  4h (or 2×2h)  ·  managers & sales  ·  global

You've already asked Claude or ChatGPT to help you prepare for a client meeting. It gave you generic questions because it had no memory of the last six months of calls, emails, and commitments. You've done the same for proposals — pasting context every time, starting from scratch every time.

This workshop builds the thing a chat box structurally cannot: a tool that accumulates. Every interaction you add grows the account's history. Every proposal you generate improves the capability library. The client context is always there. The pitch gets sharper every time.

What Will You Make

Two commands. Everything your chat tab can't do.

MODULE 1 · 2 HOURS · STANDALONE OR COMBINED

Account Intelligence

Build the tool that accumulates client history and generates structured briefs. The killer demo: brief four accounts in 90 seconds, each one referencing real interaction history.

00:00 – 00:20

Block 1 — Setup & The Thesis

Tool check. The gap between a chat tab and a tool that accumulates. Live demo: ask Claude.ai to brief you on a client — then show what the tool does instead.

Open Block
00:20 – 00:40

Block 2 — Structured Intent

Walk the PRD and its stubs. Write the first extraction rules in CLAUDE.md — live, as a group. The rules you write determine what gets captured.

Open Block
00:40 – 01:25

Block 3 — Plan, Build

Prime Claude. Plan the implementation. Build account add and account brief. 45 minutes, mostly quiet.

Open Block
01:25 – 01:40

Block 4 — Verify

Compare your brief against a hand-labelled account. Three questions: did it catch everything, did it invent anything, is it consistent? "Looks right" is not a guarantee.

Open Block
01:40 – 01:50

Block 5 — Pick Your Direction

Continue to Module 2 (Pitch Suite), or pick an extension track. Either way, leave with a clear next step and a starting prompt.

Open Block
01:50 – 02:00

Block 6 — Wrap

One sentence from each person. What changed. What's next.

Open Block
— 15-min break (combined format only) —
MODULE 2 · 2 HOURS · STANDALONE OR COMBINED

Pitch Suite

Extend the account CLI to generate tailored proposals. The capability library grows with every pitch you produce. Module 1 is a prerequisite; standalone attendees get a 10-min recap in Block 1.

00:00 – 00:10

Block 1 — Recap & Setup

Quick tool check and demo: account brief in 30 seconds. If standalone, covers the Module 1 thesis in 10 minutes.

Open Block
00:10 – 00:30

Block 2 — Pitch Intent

Walk the pitch PRD stubs. Introduce the capability library concept. Write pitch rules in CLAUDE.md — plain English, no code.

Open Block
00:30 – 01:15

Block 3 — Build the Pitch Engine

Implement account pitch. The brief seeds the proposal. The capability library prevents invented claims. 45 minutes.

Open Block
01:15 – 01:30

Block 4 — Verify the Pitch

Check the proposal against the RFP: did it address all requirements? Did it invent a capability that isn't in the library? Flag the gap, fix it in the spec.

Open Block
01:30 – 01:45

Block 5 — Full Pipeline + Tracks

Run the full pipeline: add → brief → pitch. Then pick an extension track to take home.

Open Block
01:45 – 02:00

Block 6 — Wrap

Full demo of the two-module pipeline. One sentence from each person. What changed. What to build next.

Open Block

After the Workshop — Pick a Track

Take the account tool somewhere you'd actually use it every day. Each track is an evening or a weekend.

Track A

CRM Export

account export --format salesforce outputs a CSV that imports directly into Salesforce or HubSpot. Account intelligence in the CRM, not just the CLI.

Open Track
Track B

Email Auto-Ingest

A script that watches a folder for exported .eml files (from Outlook or Gmail) and auto-runs account add. No manual trigger — the ledger stays current.

Open Track
Track C

Slack Channel Watch

An agent monitors a Slack channel where your team drops client notes and call summaries. New message → automatic processing → account ledger updated. No manual step.

Open Track
Track D

MCP + Claude Desktop

Ask Claude Desktop: "What's still open with Acme this week?" or "Which accounts haven't we contacted in 30 days?" — it queries your live ledger and answers with real data, not a guess.
