Produce a numbered, step-by-step implementation plan.

Rules:
- Read PRD.md and CLAUDE.md before planning. Do not plan from memory.
- Each step must be specific: name the file, name the function or class, describe what it does.
- Steps must be in dependency order — things that other steps rely on come first.
- End with a verification step: what command confirms the implementation is correct?
- Do not write any code. The plan is approved before any code is written.

Format:
1. [file] — [what you will do]
2. ...
N. Verify: [command to run]
